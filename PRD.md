# 鬼谷八荒地图传送增强MOD - PRD文档

## 1. 项目概述

**项目名称**: 鬼谷八荒地图传送增强MOD
**项目类型**: 游戏MOD插件
**核心功能**: 优化地图交互体验，支持按住Ctrl隐藏非城镇/宗门地图点，并实现传送符传送功能
**目标用户**: 鬼谷八荒玩家

---

## 2. 需求详述

### 2.1 功能需求

#### F1: Ctrl+地图点显示控制
- **触发条件**: 打开地图界面时，按住Ctrl键
- **功能描述**: 按住Ctrl时，隐藏除城镇和陈镇以外的所有地图点
- **释放Ctrl**: 恢复显示所有地图点

#### F2: 城镇/宗门传送检测
- **触发条件**: 点击地图上的城镇或宗门图标
- **功能描述**: 
  - 检查玩家当前是否在目标城镇/宗门范围内
  - 如果在范围内，直接触发传送

#### F3: 传送符传送功能
- **前置条件**: 玩家不在目标城镇/宗门范围内
- **功能描述**:
  - 检查玩家背包是否有对应城镇/宗门的传送符
  - 如果有传送符，弹出确认框询问是否消耗一张传送符
  - 如果没有传送符，弹出提示信息

### 2.2 数据需求

#### 传送符对应关系
| 地区 | 传送符名称 |
|------|-----------|
| 永宁州 | 永宁州传送符 |
| 雷泽州 | 雷泽州传送符 |
| 暮云州 | 暮云州传送符 |
| ... | ... |

#### 地图点类型
- **城镇/Town**: 玩家可传送的目标点
- **宗门/Faction**: 玩家可传送的目标点
- **其他点**: 资源点、事件点等

### 2.3 界面需求

#### 提示信息
- 传送符不足提示: "没有[地区名]传送符"
- 传送确认弹窗: "是否消耗1张[地区名]传送符传送到[目标名]?"

## 3. 技术方案

### 3.1 核心API使用
- `g.events`: 监听地图打开/关闭事件
- `g.world`: 获取玩家位置、宗门数据
- `g.data`: 获取背包物品数据
- `g.ui`: 弹出提示框、确认框
- Harmony补丁: 修改地图UI显示逻辑

### 3.2 模块设计

```
ModMain
├── MapPointManager       - 地图点管理（隐藏/显示控制）
├── TeleportHandler       - 传送逻辑处理
│   ├── CheckPlayerRange  - 检查玩家是否在目标范围内
│   ├── CheckItem        - 检查传送符
│   └── Teleport         - 执行传送
└── UIManager             - UI提示管理
```

## 4. 验收标准

### 4.1 功能验收
- [ ] 按住Ctrl可隐藏非城镇/宗门地图点
- [ ] 释放Ctrl恢复显示所有地图点
- [ ] 点击城镇/宗门可触发传送检测
- [ ] 范围内玩家可直接传送
- [ ] 范围外有传送符可弹窗确认
- [ ] 范围外无传送符显示提示

### 4.2 兼容性要求
- 兼容游戏当前版本
- 不影响游戏原有功能
- MOD可正常卸载（无残留数据）

---

## 5. 风险与约束

### 5.1 潜在风险
- 游戏更新可能导致API变更
- 地图UI结构变更可能影响显示控制

### 5.2 约束条件
- 需要获取背包物品数据权限
- 需要Hook地图UI相关方法
